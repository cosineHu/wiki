# 交付中心文档模板库

百胜交付方法论的标准文档模板，供项目交付团队按项目裁剪使用。包含业务调研报告模板和调研大纲模板。

## 关键信息

| 维度 | 说明 |
|------|------|
| **来源** | 百胜价值交付方法论 V3.0 |
| **模板数** | 2 个标准模板 |
| **格式** | .docx（Word 文档） |
| **用途** | 项目交付过程中的标准化文档输出 |

## 模板一：业务调研报告模板

### 基本信息

| 字段 | 内容 |
|------|------|
| 文档名称 | XX项目_业务调研报告 |
| 版本 | V1.0 |
| 出品方 | 上海百胜软件股份有限公司 |
| 适用场景 | 项目调研完成后输出正式调研报告 |

### 文档结构

```
封面（项目名称 + 版本 + 客户 + 出品方 + 日期）
├── 文档管理（文档信息/版本信息/审核批准/分发/存档）
├── 前言（感谢信 + 商业机密声明）
├── 项目概述
│   ├── 项目背景
│   ├── 调研目标（3 项）
│   └── 项目范围（3 项）
├── 业务流程（按流程逐一展开）
│   ├── 流程 A/B/C
│   │   ├── 现有作业流程（流程图占位）
│   │   ├── 流程说明
│   │   ├── 现有问题
│   │   └── 输出表单
│   └── ...
└── 期望改善点（按部门汇总）
```

### 调研目标（标准 3 项）

1. 了解目前仓库的组织架构及人员职责分工
2. 了解仓库现有作业场景及实际人员操作流程，作为系统上线后价值依据
3. 根据调研现状分析仓库现有业务流程中现存问题，形成改善点

### 流程分析模板

每个业务流程包含 4 个分析维度：

```
现有作业流程（流程图）→ 流程说明 → 现有问题 → 输出表单
```

## 模板二：调研大纲模板

### 基本信息

| 字段 | 内容 |
|------|------|
| 文档名称 | 电商业务调研大纲-XX业务调研 |
| 版本 | V1.0（2024-6-6 创建初稿，作者：刘瑶） |
| 出品方 | 上海百胜软件股份有限公司 |
| 适用场景 | 项目调研前准备标准化调研大纲 |
| 示例模块 | IT 部门调研（5 个调研问题） |

### 文档结构

```
封面（方法论标识 + 模块名称 + 版本 + 出品方）
├── 文档管理（文档信息/版本信息/审核批准/分发/存档）
├── 目录
└── 调研模块（按模块逐一展开）
    └── 每个调研问题遵循三段式结构：
        ├── 调研目的：为什么要问这个问题
        ├── 调研重点：需要关注和收集哪些信息
        └── 影响提示：调研结果对后续实施的影响
```

### 三段式调研结构

每个调研问题严格遵循：

```
调研问题（如：请介绍一下贵公司的组织架构及其职责？）
├── 调研目的：了解客户公司的组织构架以及各个部门在整个业务体系中承担的职责
├── 调研重点：组织架构、各部门职责
└── 影响提示：为后续组织架构调研做铺垫，确定调研方式（按部门/按业务线）
```

### IT 部门调研示例（5 个问题）

| # | 调研问题 | 核心关注 |
|---|---------|---------|
| 1.1 | 贵公司的组织架构及其职责？ | 组织架构、各部门职责 |
| 1.2 | 当前的信息系统架构？ | 系统架构现状、外部系统对接 |
| 1.3 | 系统架构有哪些痛点？ | 当前痛点识别 |
| 1.4 | 未来系统架构如何搭建？ | IT 规划、架构愿景 |
| 1.5 | 接下来的调研如何开展？ | 调研计划沟通 |

### 影响提示中的实施策略

| 项目类型 | 调研策略 |
|---------|---------|
| E3 升级 E3+ | 引导客户按业务召集用户快速推进，着重了解当前痛点和用户需求 |
| 友商系统更换 E3+ | 着重了解当前系统功能，结合系统演示挖掘产品差异点和需求 |

## 使用方式

### 调研报告模板

```
调研日志 → 按模板结构组织 → 填充流程图/问题/改善点 → 输出正式调研报告
```

### 调研大纲模板

```
标准调研模块（6 大模块）→ 按模板格式展开 → 裁剪定制 → 输出项目调研大纲
```

模板与 [[baisheng-survey-outline-system|调研大纲体系]] 的关系：
- 调研大纲体系定义了 **内容**（6 大模块的具体调研问题）
- 调研大纲模板定义了 **格式**（文档结构、三段式、版本管理）
