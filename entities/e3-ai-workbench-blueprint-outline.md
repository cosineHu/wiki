# E3 AI 工作台业务蓝图大纲

基于 [[e3-ai-workbench|E3 AI 工作台项目]] 需求背景和 [[blueprint-standard-library|蓝图大纲标准方案库]] 生成的项目专属业务蓝图方案。本项目为 AI 增强型项目，蓝图重点从"系统功能模块"转向"AI 能力嵌入 + 岗位工作流重塑"。

## 生成依据

| 输入 | 来源 |
|------|------|
| 需求背景 | 客户希望通过 AI 改善 E3OMS 前端 UE 交互，专岗专治降本提效 |
| 一期场景 | 预测看板 + 自动套餐 + 库存捡漏预售 + 预售审单（3.5 个场景） |
| 业务规则 | [[e3-auto-package-rules\|套餐规则]] / [[e3-presale-review-rules\|预售审单规则]] / [[e3-channel-inventory-pickup\|渠道捡漏规则]] |
| 标准方案库 | 蓝图大纲标准方案库（AI 项目裁剪适配） |

## 一、总体架构蓝图

### 1.1 系统架构

```
飞书端（轻量交互层）                    E3OMS 端（深度操作层）
┌──────────────────────┐          ┌──────────────────────────┐
│  AI 助手消息卡片       │          │  AI 工作台（嵌入式）        │
│  ├── 预测看板推送       │          │  ├── 全链路预测分析看板     │
│  ├── 预售审单通知       │  ←──→   │  ├── 自动套餐创建工作台     │
│  ├── 捡漏结果推送       │  API    │  ├── 库存捡漏+预售方案面板   │
│  ├── 异常预警           │          │  └── 预售审单工作台        │
│  └── 一键操作           │          │                           │
└──────────────────────┘          └──────────┬───────────────┘
                                             │
                              ┌──────────────┴───────────────┐
                              │        AI 能力层              │
                              │  ├── 规则引擎（套餐/捡漏/审单） │
                              │  ├── 预测模型（销量/库存趋势）  │
                              │  ├── 推荐算法（预售方案）      │
                              │  └── 数据汇总（多源聚合）      │
                              └──────────────┬───────────────┘
                                             │
                              ┌──────────────┴───────────────┐
                              │        E3OMS 数据层           │
                              │  ├── 订单中心                  │
                              │  ├── 库存中心                  │
                              │  ├── 商品中心                  │
                              │  └── 渠道数据                  │
                              └──────────────────────────────┘
```

### 1.2 双端协同设计

| 功能 | 飞书端 | E3OMS 端 |
|------|--------|---------|
| 预测看板 | 定时推送 + 异常预警卡片 | 交互式数据看板 |
| 套餐创建 | 创建结果通知 | 规则配置 + 批量创建 + 观测面板 |
| 库存捡漏 | 捡漏结果推送 + 一键确认 | 捡漏计算面板 + 历史记录 |
| 预售方案 | 方案推荐通知 | 方案配置 + 模拟测算 |
| 预售审单 | 审单通知 + 一键审核 | 审单工作台 + 异常处理 |

## 二、场景一：全链路预测分析看板

### 2.1 数据汇总层

```
多渠道订单数据 ──→ E3OMS 聚合 ──→ AI 分析引擎 ──→ 看板输出
                     │
          ┌──────────┼──────────┐
          │          │          │
       订单量      商品数     渠道分布
       趋势预测    库存预警    异常标记
```

### 2.2 看板功能矩阵

| 模块 | 数据维度 | AI 能力 | 飞书端 | E3OMS 端 |
|------|---------|--------|--------|---------|
| 订单概览 | 渠道/商品/时间 | 趋势预测 | 每日推送卡片 | 交互式图表 |
| 库存预警 | 商品/SKU/仓库 | 智能预警阈值 | 预警通知 | 预警面板 |
| 渠道对比 | 渠道/品类 | 异常检测 | 周报推送 | 对比分析 |
| 预售追踪 | 预售批次/商品 | 到货预测 | 进度通知 | 追踪面板 |

### 2.3 飞书端交互设计

```
每日 9:00 推送 → 「今日运营概览」卡片
  ├── 昨日订单量：XXX（环比 ↑5%）
  ├── 库存预警：3 个 SKU 低于警戒线
  ├── 预售到期提醒：2 个批次即将到期
  └── [查看详情] → 跳转 E3OMS 看板
```

## 三、场景二：自动套餐创建&观测工作台

### 3.1 套餐自动创建引擎

```
商品主数据（编码/名称/盒规/渠道）
    ↓
规则引擎匹配
    ├── 盒规 > 1 → 触发套餐创建
    ├── 全渠道 → ZH 整盒套餐
    ├── 得物/共鸣得物 → ZH + LG 双件套餐
    └── 抖音拆盒 → ZH + DYCH 1/2/盒规套餐
    ↓
自动生成套餐（编码 + 名称 + 规格）
    ↓
写入 E3OMS 商品中心
```

### 3.2 套餐规则配置

| 套餐类型 | 编码规则 | 命名规则 | 适用渠道 | 触发条件 |
|---------|---------|---------|---------|---------|
| ZH 整盒 | `ZH{序号}` | `整盒{商品名}` | 全渠道 | 盒规 > 1 |
| LG 双件 | `LG{序号}` | `两个{商品名}` | 得物/共鸣得物 | 盒规 > 1 |
| DYCH-1 | `DYCH{序号}1` | `1个{商品名}` | 抖音拆盒 | 盒规 > 1 |
| DYCH-2 | `DYCH{序号}2` | `2个{商品名}` | 抖音拆盒 | 盒规 > 1 |
| DYCH-N | `DYCH{序号}{盒规}` | `{盒规}个{商品名}` | 抖音拆盒 | 盒规 > 1 |

### 3.3 观测工作台

| 面板 | 展示内容 | 用途 |
|------|---------|------|
| 创建概览 | 今日/本周创建数量、成功率 | 运营监控 |
| 渠道分布 | 各渠道套餐创建占比 | 渠道分析 |
| 异常列表 | 创建失败记录及原因 | 问题排查 |
| 套餐查询 | 按商品/渠道/时间搜索套餐 | 日常查询 |

### 3.4 飞书端交互

```
新品盒规录入后 → 「套餐创建完成」卡片
  ├── 商品：DIMOO 在你身边系列（盒规12）
  ├── 已创建：ZH整盒 / LG双件 / DYCH 1个/2个/12个
  ├── 共 5 个套餐，耗时 < 10 秒
  └── [查看详情] → 跳转观测工作台
```

## 四、场景三：库存捡漏+预售方案

### 4.1 捡漏引擎

```
退款/取消事件触发
    ↓
渠道规则匹配
    ├── 微信小店：等退款审核通过 → 释放库存 → 捡漏
    ├── 抽盒机：实时退款数量 → 直接捡漏补货
    ├── 天猫：分货数 - 实际销售数 → 捡漏数量
    └── 次日统一：全部占库完成 → 按余量捡漏
    ↓
捡漏结果输出
    ├── 飞书端：推送捡漏数量 + 一键确认
    └── E3OMS 端：捡漏面板 + 历史记录
```

### 4.2 捡漏规则配置

| 渠道 | 捡漏模式 | 触发条件 | 计算方式 |
|------|---------|---------|---------|
| 微信小店 | 当天 | 退款审核通过 | 释放库存 = 捡漏库存 |
| 抽盒机 | 当天 | 实时退款 | 退款数量 = 捡漏数量 |
| 天猫 | 当天 | 订单计算完成 | 分货数 - 实际销售数 |
| 全渠道 | 次日 | 全部占库完成 | E3 实际库存余量 |

### 4.3 预售方案推荐

```
历史销售数据 + 当前库存 + 渠道特性
    ↓
AI 推荐引擎
    ├── 建议预售数量
    ├── 建议预售价格
    ├── 建议预售渠道
    └── 预计售罄时间
    ↓
预售方案输出（飞书 + E3OMS）
```

### 4.4 飞书端交互

```
捡漏完成 → 「捡漏结果」卡片
  ├── 微信小店：可捡漏 15 件
  ├── 抽盒机：可捡漏 8 件
  ├── 天猫：可捡漏 23 件
  ├── 合计：46 件
  └── [一键捡漏] [查看详情]

预售方案推荐 → 「预售建议」卡片
  ├── 建议预售：DIMOO 系列 200 件
  ├── 建议渠道：天猫 + 抖音
  ├── 预计售罄：3 天
  └── [采纳方案] [调整参数]
```

## 五、场景 3.5：预售审单自动化

### 5.1 自动化审单流程

```
预售订单数据自动汇总（提前 7 天触发）
    ↓
自动筛选（排除补发/拆盒/换货/手工/复制单）
    ↓
按用户下单时间排序
    ↓
部分到货 → 按时间序占库存
    ↓
占库存成功 → 自动审核通过
    ↓
异常处理（后单先占 → 释放 → 重新分配）
    ↓
多批次流转（剩余库存 → 下一批次）
    ↓
超 1 月预售 → 自动激活（天猫/抖音）
```

### 5.2 筛选规则配置

| 规则类型 | 规则内容 |
|---------|---------|
| 包含 | 预售订单（预售现货 + 纯预售） |
| 排除 | 补发单、在线拆盒单、换货单、手工单、复制单 |
| 排序 | 按用户下单时间升序 |
| 占库 | 先下单先占，严格时序 |
| 异常 | 后单先占 → 释放 → 重新分配 |

### 5.3 飞书端交互

```
预售提醒（提前 7 天）→ 「预售审单提醒」卡片
  ├── 预售批次：2026-06-12 批次
  ├── 待审订单：156 单
  ├── 涉及商品：DIMOO 系列 × 3 SKU
  └── [查看详情] [一键审核]

部分到货 → 「到货分配通知」卡片
  ├── 到货数量：80 件
  ├── 已自动分配：前 80 单
  ├── 待补货：76 单
  └── [查看分配明细]
```

## 六、AI 能力层设计

### 6.1 能力矩阵

| AI 能力 | 应用场景 | 技术方案 |
|---------|---------|---------|
| 规则引擎 | 套餐创建/捡漏/审单 | 业务规则配置化 + 自动执行 |
| 趋势预测 | 销量预测/库存预警 | 时序模型（移动平均/指数平滑/LSTM） |
| 异常检测 | 库存异常/订单异常 | 统计阈值 + 规则组合 |
| 智能推荐 | 预售方案推荐 | 协同过滤 + 规则约束 |
| 数据聚合 | 多源数据汇总 | ETL + 实时流处理 |

### 6.2 AI 能力部署

```
AI 能力层（独立微服务）
├── rule-engine（规则引擎）
├── predict-service（预测服务）
├── recommend-service（推荐服务）
└── data-aggregator（数据聚合）
        │
        ├──→ E3OMS API（读写）
        └──→ 飞书 API（消息推送）
```
