# Lex Fridman

全球最火的 AI 播客主持人，Lex Fridman Podcast 采访过 Elon Musk、Sam Altman、Mark Zuckerberg、[[andrej-karpathy]] 等科技领袖。在知识管理领域，他提出了独特的「运动式知识消费」模式。

## 知识管理方法

Lex 与 Karpathy 类似，也使用 LLM 驱动的个人知识库系统，但有三个独特之处：

### 前端混合体

Obsidian + Cursor（编辑 .md 文件）+ 自己 vibe-coded 的网页终端。不满足于 Markdown 的静态渲染——经常让 LLM 生成动态 HTML 页面（带 JavaScript），实现交互式数据排序、筛选和可视化调整。

### 播客驱动的知识需求

因为播客嘉宾覆盖领域极广，研究兴趣数量和多样性远超普通人。这种场景下知识库方法的优势更明显——不可能把所有领域的知识都记在脑子里。

### 跑步语音播客

最炸裂的用法：让系统为特定主题生成临时迷你知识库，加载进 LLM，然后在 7-10 英里的长跑中开启语音模式。跑步时变成交互式播客——随时提问、听答案、深入探讨。知识摄入方式从「坐着读」变成「边跑边聊」。

## 核心哲学

「知识要可交互」——不满足于静态阅读，追求动态可视化、语音对话、运动中学习。
