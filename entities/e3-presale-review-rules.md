# 预售审单规则

预售订单处理智能化平台的业务规则，覆盖数据汇总、订单筛选、部分到货处理、渠道激活等全流程。

## 一、建设背景

当前预售订单处理依赖人工导表、人工核对及跨部门沟通，存在以下问题：
- 客服需提前人工导出预售订单
- 商品人员需人工确认到货情况
- 多平台数据需二次校准
- 人工筛选和排查效率较低
- 部分到货场景需人工释放库存及重新审核

## 二、数据汇总规则

### 2.1 预售订单数据自动汇总

通过 E3 自动汇总预售订单数据并生成看板，展示维度：
- 渠道
- 商品
- 预计预售发货时间
- 订单量
- 商品数

### 2.2 预售任务自动触发

| 规则 | 说明 |
|------|------|
| 触发时机 | 订单预计发货时间临近时 |
| 提前天数 | 7 天 |
| 通知对象 | 客服、商品人员 |

## 三、预售订单筛选规则

### 3.1 数据查询来源

需同时通过**平台**和 **E3** 两个渠道获取数据并进行二次校准：
- 核实前端展示的真实预售时间
- 存在不同预售模式：预售现货、纯预售

### 3.2 需排除订单类型

以下订单不纳入预售处理范围：
- 补发单
- 在线拆盒单
- 换货单
- 手工单
- 复制单

## 四、部分到货处理规则

### 4.1 库存占用原则

> **按用户下单时间顺序占库存**

- 先下单订单优先占库存
- 占库存成功后系统自动审核通过

### 4.2 库存顺序异常处理

若后下单订单先占库存，则：
1. 释放后单库存
2. 重新分配给前序订单

> 必须保证库存分配严格遵循下单时间顺序

### 4.3 多批次预售处理

- 前一批次审核完成后，如存在剩余库存
- 剩余库存按顺序继续分配给下一预售批次
- 库存需跨批次按时间顺序流转

## 五、渠道预售激活规则

| 维度 | 说明 |
|------|------|
| 涉及渠道 | 天猫、抖音（其他渠道预售订单量较少） |
| 触发规则 | 只要产生预售任务，提前 7 天执行激活检查 |
| 激活范围 | 仅针对超过一个月的预售订单 |

## 六、当前人工处理链路

```
客服导表 → 商品确认到货 → 人工筛选订单 → 人工占库存 → 人工释放异常库存 → 人工审核
```

整体人工成本较高，AI 改造目标为全链路自动化。
