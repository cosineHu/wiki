# E3 自动创建套餐规则

基于商品盒规自动生成套餐的规则体系，覆盖全渠道、得物渠道、抖音拆盒渠道三类场景。

## 一、整盒套餐（全渠道通用）

| 维度 | 说明 |
|------|------|
| **用途** | 基于商品盒规自动生成整盒套餐，盒规>1 的商品 |
| **适用范围** | 全渠道统一使用 |
| **生成规则** | 按商品盒规自动创建对应套餐，套餐数量 = 商品盒规 |
| **编码规则** | `ZHXXXX` |
| **命名规则** | `整盒XXX` |

**示例**：商品 DIMOO 在你身边系列，盒规 12 → 编码 ZH123456 → 名称"整盒DIMOO在你身边系列"

## 二、得物渠道双件套餐

| 维度 | 说明 |
|------|------|
| **适用渠道** | 得物、共鸣得物 |
| **用途** | 盒规>1 的商品，在整盒套餐基础上额外生成 2 个装套餐 |
| **编码规则** | `LGXXXX` |
| **命名规则** | `两个XXX` |

**示例**：编码 LG123456 → 名称"两个DIMOO在你身边系列"

## 三、抖音拆盒渠道套餐

| 维度 | 说明 |
|------|------|
| **适用渠道** | 抖音拆盒（DYCH）：抖音旗舰店、抖音盲盒专卖店 |
| **用途** | 盒规>1 的商品，自动生成 1个装/2个装/整盒装 |
| **编码规则** | `DYCHXXXX1` / `DYCHXXXX2` / `DYCHXXXX{盒规}` |
| **命名规则** | `1个XXX` / `2个XXX` / `{盒规}个XXX` |

**示例**（盒规12）：DYCH1234561→1个DIMOO、DYCH1234562→2个DIMOO、DYCH12345612→12个DIMOO

## 四、自动创建逻辑汇总

系统需根据商品主数据自动识别：
- 商品编码
- 商品名称
- 商品盒规
- 渠道配置

并自动生成：
1. **ZH** 整盒套餐（全渠道）
2. **LG** 双件套餐（得物渠道）
3. **DYCH** 1/2/盒规套餐（抖音拆盒渠道）
