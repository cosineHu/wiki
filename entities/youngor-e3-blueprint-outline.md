# 雅戈尔 E3+ 业务蓝图大纲

## 生成依据

| 输入 | 来源 |
|------|------|
| 调研日志 | 雅戈尔 E3+ 调研日志（11 Sheet，1055条需求，89条接口） |

| 项目 | 说明 |
|------|------|
| 供应商 | 供应商档案管理 |
| 客户 | 经销商/分销商档案 |

| 仓库 | 物理仓 + 逻辑仓，匹配 E3+ 四层库存模型 |

#### 1.1.3 档案中心对接流程 ✅

- 与用友中台的主数据同步方案
- 档案变更的增量同步机制

### 1.2 商品主数据

#### 1.2.1 属性定义 ✅

| 属性类型 | 雅戈尔定制 |
|---------|-----------|
| 基础属性 | 品名、品牌、年份季节、波段 |
| 销售属性 | 尺码组（服装鞋帽）、颜色组 |
| 扩展属性 | 面料、版型、风格、适用场景 |
| 辅助属性 | 吊牌价、成本价、重量、体积 |

#### 1.2.2 物料类别 ✅

- 商品类别：上衣/裤装/裙装/外套/配饰/鞋类
- 商品类目：按品牌→年份→季节→系列→品类 五级分类

#### 1.2.3 物料管理 ✅

| 项目 | 说明 |

### 1.3 用户中心 ✅

#### 1.3.1 用户及权限管理架构

- 集团→法人公司→部门 三级权限隔离
- 角色：系统管理员/业务主管/运营/仓管/财务/门店

#### 1.3.2 部门/岗位/员工/账户管理

#### 1.3.3 功能权限 + 数据权限

- 功能权限：按角色分配菜单和操作权限
- 数据权限：按组织层级隔离数据（集团/法人/门店）

## 二、库存管理蓝图

### 2.1 采购管理 ✅

#### 2.1.1 采购入库

- 采购单创建→收货→质检→入库→记账
- 与道讯 DRP 的采购数据对接

#### 2.1.2 采购退货

- 退货申请→审核→出库→供应商确认

### 2.2 库存管理 ✅

#### 2.2.1 库存调拨

| 场景 | 说明 |
|------|------|
| 仓间调拨 | 总仓→区域仓→门店仓 |
| 门店间调拨 🆕 | 邻店调拨，减少调拨费用（售前目标：节省 100 万/年） |
| 渠道间调拨 | 线上库存↔线下库存 灵活调配 |

#### 2.2.2 库存调整

- 盘点调整、报损报溢、库存修正

#### 2.2.3 库存移仓

- 物理仓内库位移动

#### 2.2.4 库存锁定 🆕

| 锁定类型 | 说明 |
|---------|------|
| 订单锁定 | 下单即锁定，防止超卖 |
| 活动锁定 | 大促活动预留库存 |
| 质检锁定 | 退货质检期锁定 |

### 2.3 库存同步 ✅

#### 2.3.1 平台商品关联

- E3+ SKU ↔ 各平台商品 ID 映射

#### 2.3.2 库存同步架构 🆕（雅戈尔定制）

```
当前：POS → 中台 → 差异化更新 → E3（全量同步，每晚21点）
目标：E3+ 库存中心（统一库存视图）
  ├── 全渠道共享（按可用数 + 警戒数 + 保留数）
  ├── 实时增量同步（门店库存变动实时推送）
  └── 单据对接模式（替代刷库存模式）
```

| 同步策略 | 说明 |
|---------|------|
| 共享模式 | 全渠道共享库存池，按可用数同步 |
| 独享模式 | 特定渠道独占库存（如唯品会档期库存） |
| 警戒数 | 库存低于警戒数时触发预警/停止同步 |
| 保留数 | 为特定渠道/活动保留的库存数量 |

## 三、电商订单管理蓝图

### 3.1 B2C 销售业务 ✅

#### 3.1.1 订单履约主流程 🆕（雅戈尔定制）

```
平台订单下载 → 格式化 → 自动化审核 → 仓库适配 → 拆单策略
    → 快递适配 → 免审规则 → 推单发货 → 出库回传 → 回写平台
```

| 节点 | 雅戈尔定制说明 |
|------|--------------|
| 订单下载 | 淘宝/京东/唯品会/抖音/快手/自研商城/酷客多/喵街/天虹/杉杉奥莱 |
| 自动化审核 | 地址校验、库存校验、风控规则、赠品匹配 |
| 仓库适配 | 就近仓库/库存优先/成本优先 多策略 |
| 拆单策略 | 按仓库拆单、按预售拆单、按大件拆单 |
| 快递适配 | 按重量/地区/时效自动匹配快递 |
| 免审规则 | 低风险订单自动审核通过 |

#### 3.1.2 订单下载 ✅

- 各平台订单下载频率和方式
- 下载异常重试机制

#### 3.1.3 促销策略 🆕

| 促销类型 | 说明 |
|---------|------|
| 满减/满赠 | 订单级促销计算 |
| 优惠券 | 平台券/店铺券叠加规则 |
| 限时折扣 | 秒杀/闪购价格管理 |
| 赠品策略 | 买赠/满赠送规则 |

#### 3.1.4 订单合并/拆分 ✅

- 同地址同收货人自动合并
- 异仓/异品牌/预售商品自动拆分

#### 3.1.5 仓库适配 ✅

- 库存优先（选库存最多的仓）
- 距离优先（选离收货地址最近的仓）
- 成本优先（选物流成本最低的仓）

#### 3.1.6 快递适配 ✅

- 快递公司优先级配置
- 按重量/体积/地区匹配

#### 3.1.7 免审规则 ✅

- 低金额订单免审
- 老客户订单免审
- 特定渠道免审

#### 3.1.8 推单发货 🆕

| 渠道 | 推单方式 |
|------|---------|
| 门店发货 | E3+ → POS（替代 E3→中台→POS 三步） |
| WMS 发货 | E3+ → WMS 直连 |
| 第三方仓 | E3+ → 第三方 WMS |

#### 3.1.9 出库回传 ✅

- WMS/POS 出库确认回传
- 快递单号回传

#### 3.1.10 回写平台 ✅

- 发货状态回写各电商平台
- 快递信息回写

### 3.2 B2C 售后业务 ✅

#### 3.2.1 订单取消 🆕

| 取消场景 | 处理逻辑 |
|---------|---------|
| 未发货取消 | 自动取消，释放库存 |
| 已发货取消 | 拦截快递，或转为退货流程 |
| 零售订单强制取消 | 门店 POS 发起取消，E3+ 同步处理 |

#### 3.2.2 退货退款 ✅

```
退单下载 → 匹配原单 → 退货地址分配 → 收货入库 → 质检 → 退款
```

#### 3.2.3 换货业务 ✅

- 换货申请→审核→发新货→收旧货
- 换货库存处理（先发后收/先收后发）

#### 3.2.4 退货包裹单处理 🆕

- 自动下载退换货申请
- 包裹匹配（退货运单号↔原订单）
- 包裹签收确认

### 3.3 B2B 销售业务 ✅

#### 3.3.1 唯品会 JIT 业务 ✅

| 模式 | 说明 |
|------|------|
| JIT | 按订单生产/备货，48 小时内发货 |
| JITX | 极速 JIT，24 小时内发货 |
| 档期管理 | 档期创建→商品报名→库存锁定→发货→结算 |

#### 3.3.2 京东自营批发业务 ✅

- 京东自营采购单→发货→结算

### 3.4 B2B 售后业务 ✅

#### 3.4.1 唯品会退供

- 退供单处理→退货入库→结算调整

#### 3.4.2 京东自营退货

- 京东退货单→退货入库→结算调整

## 四、全渠道业务蓝图

### 4.1 全渠道订单分单策略 🆕（雅戈尔定制）

| 分单策略 | 说明 |
|---------|------|
| 普通订单分单 | 按库存/距离/成本综合评分 |
| 分销订单分单 | 按经销商区域/库存分配 |
| 即时零售分单 | 按门店距离/配送时效（30分钟/1小时达） |
| 预售订单分单 | 按预售仓/发货时间分流 |

### 4.2 网订店发 ✅

```
线上订单 → 分单到门店 → 门店接单 → 门店发货 → 快递配送
```

| 节点 | 雅戈尔定制 |
|------|-----------|
| 门店接单 | 门店 APP/终端接单提醒，超时自动转单 |
| 拒单处理 | 门店拒单后自动重路由到其他门店/仓库 |
| 时效要求 | 接单 15 分钟，发货 2 小时 |

### 4.3 店订店发 ✅

- 门店缺货时，代客下单从其他门店/仓库发货

### 4.4 店订仓发 ✅

- 门店下单，仓库直发客户

### 4.5 全渠道售后 ✅

- 线上订单门店退换货
- 门店订单线上退换货

## 六、技术架构蓝图 🆕（雅戈尔定制）

> 此模块为标准方案库之外的新增模块，基于雅戈尔 IT 现状定制。

### 6.1 系统架构

```
TO-BE 架构：
各电商平台/自研商城/门店POS
    ↓
百胜 Adaptor（全网通）
    ↓
E3+ 业务中台
├── 订单中心（自动化审核/灵活适配/异常处理）
├── 库存中心（四层库存/共享独享/实时同步）
├── 商品中心（属性管理/物料管理）
├── 财务中心（自动对账/收入确认/财务报表）
└── 用户中心（权限管理/数据隔离）
    ↓
外部系统对接
├── WMS（仓库管理系统）
├── POS（门店销售系统）
├── 用友中台（主数据同步，逐步替代）
├── 道讯 DRP（分销管理）
└── 财务 ERP（凭证对接）
```

### 6.2 接口清单（89 条）

| 外部系统 | 接口数 | 典型接口 |
|---------|--------|---------|
| 微商城 | 9+ | 商品列表、库存同步、订单推送、售后 |
| WMS | 5+ | 订单推送、出库回写、取消、唯品会发货 |
| POS | 3+ | 订单取消、发货回写 |
| 用友中台 | 多个 | 主数据同步 |

### 6.3 迭代计划（5 Sprint）

| Sprint | 周期 | 内容 |
|--------|------|------|
| Sprint 1 | 2 周 | 分单策略 + 主数据接口 |
| Sprint 2 | 2 周 | 分单策略 + WMS 对接 + 唯品会 |
| Sprint 3 | 2 周 | 财务对账 + POS/微商城接口 |
| Sprint 4 | 2 周 | 收尾优化 |
| Sprint 5 | 2 周 | 发布 |

**总工期**：约 10 周

## 裁剪说明

相比标准方案库，雅戈尔 E3+ 蓝图大纲的定制化调整：

| 调整类型 | 内容 |
|---------|------|
