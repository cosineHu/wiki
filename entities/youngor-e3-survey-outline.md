# 雅戈尔 E3+ 调研大纲

基于 [[youngor-e3-presale|雅戈尔 E3+ 售前方案]] 和 [[baisheng-survey-outline-system|百胜调研大纲体系]] 生成的项目专属调研大纲。遵循"调研目的 → 调研重点 → 影响提示"三段式结构。

## 生成依据

| 输入 | 来源 |
|------|------|
| 售前资料 | 雅戈尔 E3+ 售前方案（68页PPT），AS-IS 架构：E3 OMS + 用友中台 + 道讯 DRP/POS + WMS |
| 标准方案库 | 百胜调研大纲体系（6大模块） |
| 核心机会点 | 自研商城业务快速扩展、全渠道业务高效协同 |

## 一、IT 部门调研

> 了解客户 IT 基础设施和系统架构，为 E3+ 升级提供技术基础。

### 1.1 系统架构现状

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认当前系统架构全景，明确 E3+ 替换/对接范围 |
| **调研重点** | E3 OMS 版本及功能范围、用友中台承担的角色（路由/主数据/库存分配）、道讯 DRP/POS/WMS 的接口方式 |
| **影响提示** | 决定 E3+ Adaptor 全网通的对接策略，影响接口清单和迁移方案 |

**售前已知**：
- 酷客多/喵街/天虹/杉杉奥莱 → 用友中台 → E3 OMS
- 淘宝/京东/唯品会/抖音/快手 → E3 OMS
- 线下门店 → POS → 中台 → E3

**待确认**：
- [ ] 各外部系统的接口协议（API/DB/文件）和对接方式
- [ ] 用友中台是否保留？保留哪些功能？
- [ ] 是否有自研商城？技术栈和接口能力？

### 1.2 组织架构与职责

| 维度 | 内容 |
|------|------|
| **调研目的** | 了解 IT 部门组织架构，明确对接人和决策链 |
| **调研重点** | IT 团队规模、分工（开发/运维/业务分析）、决策流程 |
| **影响提示** | 影响项目沟通计划和需求确认流程 |

### 1.3 系统痛点与期望

| 维度 | 内容 |
|------|------|
| **调研目的** | 挖掘 IT 部门的核心痛点，确认技术升级的驱动力 |
| **调研重点** | 性能瓶颈（售前已知 E3 架构老旧）、运维成本、扩展性需求 |
| **影响提示** | 影响技术方案设计（云原生/微服务/性能指标） |

**售前已知痛点**：
- 部分渠道经用友中台中转，链路长、时效低
- E3 以流程自动化为中心，缺乏服务化能力

---

## 二、基础主数据调研

> 梳理组织、库存、配送等基础配置数据，为 E3+ 四层库存架构落地做准备。

### 2.1 组织架构

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认雅戈尔集团化多组织架构，匹配 E3+ 法人公司→核算组织→库存组织→逻辑仓模型 |
| **调研重点** | 法人实体数量、核算组织层级、各组织间的货权关系 |
| **影响提示** | 决定 E3+ 组织建模方案，影响库存共享/独享策略设计 |

**售前已知**：雅戈尔为集团化多组织架构，需支持法人公司→核算组织→库存组织→逻辑仓。

**待确认**：
- [ ] 各法人公司/核算组织的具体清单
- [ ] 是否存在跨法人调拨场景

### 2.2 库存架构

| 维度 | 内容 |
|------|------|
| **调研目的** | 梳理物理仓库和逻辑仓结构，匹配 E3+ 四层库存逻辑 |
| **调研重点** | 物理仓数量/位置/类型、逻辑仓划分规则、当前库存管理方式 |
| **影响提示** | 决定库存中心的设计方案（物理库存→逻辑库存→唯一码库存） |

**售前已知**：当前按仓数同步，两系统交互；目标按可用数同步，统一库存中心。

**待确认**：
- [ ] 物理仓清单（含第三方仓）
- [ ] 当前库存分配规则（中台按比例分配到各渠道）
- [ ] 是否需要唯一码（SKU 级/件级）管理

### 2.3 配送方式

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认快递/物流配送方式配置 |
| **调研重点** | 合作快递公司、配送区域覆盖、时效要求 |
| **影响提示** | 影响订单履约中的快递适配策略 |

### 2.4 营销区域与标签

| 维度 | 内容 |
|------|------|
| **调研目的** | 了解区域划分和标签体系 |
| **调研重点** | 营销区域划分规则、商品/客户标签体系 |
| **影响提示** | 影响分单策略和营销活动配置 |

## 三、商品主数据调研

> 明确商品属性定义、物料分类和档案管理，为 E3+ 商品中心建设打基础。

### 3.1 属性定义

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认商品属性体系（基础/销售/扩展/辅助属性） |
| **调研重点** | 雅戈尔服装行业的特殊属性（尺码、颜色、季节、系列等） |
| **影响提示** | 影响商品中心的属性模型设计 |

**待确认**：
- [ ] 服装行业特有属性：尺码组、颜色组、波段、年份季节
- [ ] 是否有多规格（SKU 组合）管理需求

### 3.2 物料类别与管理

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认商品分类体系和物料管理方式 |
| **调研重点** | 商品类别/类目层级、SKU 编码规则、条码管理 |
| **影响提示** | 影响商品档案结构和数据迁移方案 |

---

## 四、库存管理调研

> 确定库存架构、对接方式和同步策略，为 E3+ 库存中心设计提供依据。

### 4.1 库存对接方式

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认当前库存对接方式，评估升级方案 |
| **调研重点** | 刷库存模式 vs 单据对接模式、各系统的库存同步频率和方式 |
| **影响提示** | 决定 E3+ 库存中心的对接策略 |

**售前已知**：
- 全量同步：每晚 21 点 POS → 中台 → 差异化更新 → E3
- 增量同步：门店库存变动实时推送
- 中台按比例分配库存到各渠道

**待确认**：
- [ ] 是否从"刷库存"升级为"单据对接"模式
- [ ] 各渠道库存同步频率和时效要求

### 4.2 库存共享策略

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认全渠道库存共享策略 |
| **调研重点** | 共享模式（按可用数+警戒数+保留数）vs 独享模式、各渠道库存分配规则 |
| **影响提示** | 影响库存中心的共享/独享策略配置 |

### 4.3 报表需求

| 维度 | 内容 |
|------|------|
| **调研目的** | 了解当前库存报表使用情况和新增需求 |
| **调研重点** | 现有报表清单、报表生成频率、数据维度需求 |
| **影响提示** | 影响报表开发和数据看板设计 |

---

## 五、电商业务调研

> 梳理 2C/2B/O2O 全业务流程，为 E3+ 订单中心设计提供依据。

### 5.1 2C 正向订单

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认各平台正向订单处理流程 |
| **调研重点** | 订单下载/格式化/审核/适配/发货全链路、促销策略、合并/拆分规则、仓库/快递适配、免审规则 |
| **影响提示** | 影响订单中心的自动化审核和灵活适配策略设计 |

**售前已知**：
- 路由在用友中台完成，E3 不做自动合并/拆分
- 目标：E3+ 自动化审核 + 灵活适配策略

**待确认**：
- [ ] 各平台（淘宝/京东/唯品会/抖音/快手/自研商城）的订单量级和峰值
- [ ] 当前促销策略（满减/满赠/优惠券/限时折扣）
- [ ] 订单合并/拆分规则（同地址合并？异仓拆单？）
- [ ] 快递适配规则（按重量/地区/时效选快递）

### 5.2 2C 逆向流程

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认退换货处理流程 |
| **调研重点** | 订单取消、退货退款、换货业务、售后单据流转 |
| **影响提示** | 影响逆向链路设计（自动下载退换货申请、包裹匹配） |

**售前已知**：退货退款流程涉及多个系统协同，换货打标处理。

**待确认**：
- [ ] 退货地址管理规则
- [ ] 退款审批流程和权限
- [ ] 换货的库存处理逻辑

### 5.3 O2O 业务

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认线上线下协同业务场景 |
| **调研重点** | 线上订单门店发货/自提、门店退换货、全渠道分单策略 |
| **影响提示** | 影响全渠道业务模块设计 |

**售前已知**：门店发货需经 E3→中台→POS 三步，目标优化为更短链路。

**待确认**：
- [ ] 门店自提的库存扣减时机
- [ ] 门店接单/拒单的时效要求
- [ ] 全渠道分单策略（就近门店？库存优先？）

### 5.4 唯品会业务

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认唯品会 JIT/JITX 业务模式 |
| **调研重点** | JIT/JITX 模式差异、档期管理、发货时效要求 |
| **影响提示** | 影响唯品会对接模块设计 |

### 5.5 2B 业务

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认分销/经销业务流程 |
| **调研重点** | 分销订单流程、代发业务、经销商管理 |
| **影响提示** | 影响 2B 订单模块设计 |

---

## 六、电商对账业务调研

> 确定对账流程和财务报表需求，为 E3+ 财务中心设计提供依据。

### 6.1 对账现状

| 维度 | 内容 |
|------|------|
| **调研目的** | 了解当前对账流程和痛点 |
| **调研重点** | 当前对账方式（手工/系统）、对账频率、差异处理方式 |
| **影响提示** | 决定自动对账方案的设计重点 |

**售前已知**：账单格式不统一，手工导入，差异难追溯。

**待确认**：
- [ ] 各平台账单格式和获取方式
- [ ] 当前对账周期（日/周/月）
- [ ] 差异处理的审批流程

### 6.2 对账规则

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认对账规则和收入确认方式 |
| **调研重点** | 资金对账/销售对账规则、收入确认时点（发货/签收）、账套和会计期间 |
| **影响提示** | 影响对账规则配置和财务报表设计 |

### 6.3 财务报表

| 维度 | 内容 |
|------|------|
| **调研目的** | 确认财务报表需求 |
| **调研重点** | 资金三栏账、商店资金对账、自营商店结算单等报表需求 |
| **影响提示** | 影响财务报表开发范围 |

## 调研执行计划

| 模块 | 预计时长 | 参与角色 | 优先级 |
|------|---------|---------|--------|
| IT 部门调研 | 0.5 天 | IT 负责人、架构师 | P0 |
| 基础主数据 | 1 天 | 业务主管、IT | P0 |
| 商品主数据 | 0.5 天 | 商品管理、IT | P1 |
| 库存管理 | 1 天 | 仓储物流、IT | P0 |
| 电商业务 | 2 天 | 电商运营、客服 | P0 |
| 电商对账 | 1 天 | 财务、IT | P1 |

**总预计**：6 天（可并行压缩至 4 天）
