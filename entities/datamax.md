# DataMax

DataMax 数据中台，基于云上或私有化轻数仓平台部署，帮助客户对数据资产统一管理、人货场运营分析及驱动业务数据化运行，支撑企业实现数据共享、精准营销、辅助决策等应用业务场景，助力企业数字化零售转型。

## 产品概述

| 维度 | 说明 |
|------|------|
| **产品定位** | 数据中台（轻量级数仓应用方案） |
| **部署方式** | 云上部署 / 私有化部署 |
| **核心能力** | 数据资产统一管理、人货场运营分析、业务数据化驱动 |
| **数据规模** | 亿级数据量秒查 |
| **应用场景** | 数据共享、精准营销、辅助决策、数字化零售转型 |

## 产品设计理念

> **数据像积木一样，自由组合，驱动业务创新。**

- **轻量级数仓应用方案**：不依赖重型数仓基础设施，快速落地
- **人、货、场标签体系**：画像应用及扩展能力，支撑精细化运营
- **亿级数据秒查**：架构支撑海量数据实时查询
- **数据低代码平台**：提供人、货、数据集成快速开发能力，降低技术门槛

## 重点建设方向：商品运营

DataMax 当前重点建设**商品运营**领域，围绕"智能配补调"场景构建了**商品智能体（Merchandise Agent）**。

### 商品智能体

| 维度 | 说明 |
|------|------|
| **核心场景** | 智能配补调（配货/补货/调拨/退货） |
| **能力层级一** | 业务运营：规则计算、单据生成、库存可视化 |
| **能力层级二** | 智能问数：基于商品数据和业务规则的自然语言数据查询 |
| **能力层级三** | 数据分析：多维度商品运营分析、趋势洞察、决策建议 |

> 借助 Agent 能力，DataMax 从"被动执行规则"升级为"主动分析建议"，能力不止于业务运营，扩展到智能问数和数据分析。

## 客户画像

- **行业**：不限（鞋服、零售、快消等多行业适用）
- **能力要求**：有开发能力、业务运营能力强
- **数据基础**：私域数据全面效果更优
- **典型客户**：广州尚睿服装（配补调退业务）

## 核心应用：配补调退

基于 DataMax 数据中台能力构建的商品**配货、补货、调拨、退货**全流程管理应用。详见 [[replenishment-allocation-transfer-return|配补调退业务]]。

### 补货模块

基于安全库存和近 7 天销量的智能补货计算：

- **安全库存**：上市≤7 天 SKU = 上市库存；上市＞7 天 SKU = 每周一按近 7 天实际销量更新
- **目标库存** = 近 7 天平均销量 × X（可配置）
- **应补数量** = 目标库存 - 在仓库存 - 在途库存 - 配未发库存
- **仓库优先级**：总仓 > 补货仓 > 门店仓

### 调拨模块

断码调拨，每周 1 次：

- **调出优先级**：30 日无销售 > 14 日无销售 > 7 日无销售
- **店铺等级排序**：B > A > A+ > S > S+（C 店不参与）
- **安全库存动态调整**：7-14 天无销 → 1 件；21 天无销 → 0

### 配货模块

两种场景：
- **期货配货**：按订单配货，门店优先级 S+ > S > A+ > A > B > C，分波段批量配货
- **增补款配货**：按 SKC 尺码比 + 配货手数分配库存

### 退货模块

- **频率**：每周三执行
- **筛选维度**：30/45/60 天不动销、年份、波段
- **加盟店**走渠道调拨单，**直营店**走商店退货申请单

## 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 单模块筛选+计算 ≤3 秒（10 万条）；批量生成 ≤100 条单据 ≤5 秒 |
| 易用性 | 操作流程可视化（筛选→计算→确认→生成单据） |
| 安全 | 角色权限控制、单据自动备份 |
