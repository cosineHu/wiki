# 交付中心知识库三大核心场景

基于交付中心知识库的标准化交付流程，定义三个核心场景及其依赖关系。

## 场景依赖链

```
售前资料（按项目导入）
    │
    ├──→ 场景一：生成调研大纲 ──→ 调研大纲（按项目输出）
    │        依赖：标准方案库（调研大纲体系）
    │
    ├──→ 场景二：生成调研日志 ──→ 调研日志（按项目输出）
    │        依赖：调研大纲 + 调研会议纪要
    │
    └──→ 场景三：生成业务蓝图大纲 ──→ 业务蓝图方案（按项目输出）
             依赖：标准方案库（蓝图大纲体系） + 调研日志/报告
```

## 场景一：生成调研大纲

### 输入

| 输入项 | 说明 | 示例 |
|--------|------|------|
| 售前资料 | 按项目导入的售前方案 PPT/文档 | 雅戈尔 E3+ 售前方案（68页PPT） |
| 标准方案库 | 交付中心知识库中的调研大纲体系 | [[baisheng-survey-outline-system\|百胜调研大纲体系]]（6大模块） |

### 处理逻辑

```
售前资料分析（AS-IS/TO-BE/痛点/机会点）
    ↓
匹配标准调研模块（6大模块：IT/主数据/商品/库存/电商/对账）
    ↓
裁剪 + 定制化（根据客户现状补充专项调研问题）
    ↓
输出项目专属调研大纲（三段式：目的→重点→影响提示）
```

### 输出

项目专属调研大纲，参见 [[youngor-e3-survey-outline|雅戈尔 E3+ 调研大纲]]。

## 场景二：生成调研日志

### 输入

| 输入项 | 说明 | 示例 |
|--------|------|------|
| 调研大纲 | 场景一输出的项目专属调研大纲 | 雅戈尔 E3+ 调研大纲 |
| 调研会议纪要 | 按项目导入的调研会议记录 | 各模块调研会议纪要 |

### 处理逻辑

```
调研大纲（结构化问题列表）
    ↓
逐项调研 → 记录：业务现状 / 涉及系统 / 提升方向 / 备注
    ↓
汇总 → 输出调研日志（结构化记录）
```

### 输出

项目调研日志，参见 [[youngor-e3-survey-log|雅戈尔 E3+ 业务调研日志]]。

## 场景三：生成业务蓝图大纲

### 输入

| 输入项 | 说明 | 示例 |
|--------|------|------|
| 标准方案库 | 交付中心知识库中的蓝图大纲体系 | [[blueprint-standard-library\|蓝图大纲标准方案库]]（6大分类） |
| 调研日志/报告 | 场景二输出的调研日志 | 雅戈尔 E3+ 调研日志（1055条需求） |

### 处理逻辑

```
调研日志分析（AS-IS 现状 + TO-BE 提升方向）
    ↓
匹配标准蓝图模块（6大分类：主数据/库存/电商订单/全渠道/财务对账）
    ↓
裁剪 + 定制化（根据调研发现增删模块、调整方案细节）
    ↓
输出项目专属业务蓝图大纲（四级树形结构）
```

### 输出

项目业务蓝图大纲，参见 [[youngor-e3-blueprint-outline|雅戈尔 E3+ 业务蓝图大纲]]。

## 场景四：生成测试用例

### 输入

| 输入项 | 说明 | 示例 |
|--------|------|------|
| 标准方案库 | 交付中心知识库中的 SIT 测试用例体系 | [[sit-test-case-library\|SIT 测试用例标准方案库]]（7 Sheet，~50 个用例） |
| 业务蓝图大纲 | 场景三输出的项目业务蓝图方案 | 雅戈尔 E3+ 业务蓝图大纲（6大分类） |

### 处理逻辑

```
业务蓝图大纲分析（识别需测试的功能模块）
    ↓
匹配标准测试用例（5大模块：2C订单/2B订单/全渠道/库存/对账）
    ↓
裁剪 + 定制化（按项目平台/系统/业务场景裁剪）
    ↓
输出项目专属 SIT 测试用例（13 字段标准结构）
```

### 输出

项目 SIT 测试用例，参见 [[youngor-e3-sit-test-cases|雅戈尔 E3+ SIT 测试用例]]（56 个用例，6 大模块）。

```
│   └── [[youngor-e3-survey-outline|雅戈尔 E3+ 调研大纲]]         ← 场景一输出 → 场景二输入

└── 测试用例（按项目）
    └── [[youngor-e3-sit-test-cases|雅戈尔 E3+ SIT 测试用例]]     ← 场景四输出
```
