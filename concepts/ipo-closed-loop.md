# IPO 闭环

## 定义

IPO 闭环（Input-Process-Output Loop）是场景驱动认知闭环中将"描述性知识"转化为"可执行规范"的关键机制。每个原子概念被建模为一段完整的流程：需要什么输入(Input)、内部经过哪些步骤(Process)、产出什么结果(Output)、过程中用到哪些实体作为工具(Tools)。

## IPO 四元组

```
┌──────────┐    ┌──────────────┐    ┌──────────┐
│  Input   │ →  │   Process    │ →  │  Output  │
│ 输入条件  │    │  处理步骤     │    │  产出结果  │
└──────────┘    └──────┬───────┘    └──────────┘
                       │
                  ┌────▼────┐
                  │  Tools   │
                  │ 工具实体  │
                  └─────────┘
```

### Input（输入）

- 该方法启动所需的前置条件
- 需要什么数据、信息、上下文
- 示例：第一性原理的输入是"待分析的复杂问题"

### Process（处理步骤）

- 内部经过哪些操作步骤
- 每步的转换逻辑
- 示例：识别表面现象 → 剥离类比假设 → 追问底层要素 → 基于第一原理推导

### Output（输出）

- 该方法产出的结果
- 输出的格式和特征
- 示例：基于底层逻辑的解决方案

### Tools（工具实体）

- 过程中调用的实体（工具、模型、框架）
- 链接到实体层的具体条目
- 示例：SBR 模型、归纳-演绎闭环

## 为什么需要 IPO

### 从"是什么"到"怎么用"

传统知识管理描述方法的"定义"和"特征"，但 AI 需要的是"在什么条件下、经过什么步骤、产出什么结果"。IPO 把方法从叙述性文字转化为可执行的流程规范。

### 可组装性

只有每个方法都有明确的输入输出接口，才能像搭积木一样把它们串接成组合概念和场景。没有 IPO 的方法就像没有函数签名的代码——无法被可靠地调用和组合。

### 可验证性

IPO 提供了验证标准：给定正确的输入，经过规定的处理步骤，是否产出了预期的输出？这让方法的有效性可以被测试和迭代。

## 示例：第一性原理的 IPO

| 维度 | 内容 |
|------|------|
| Input | 待分析的复杂问题（问题陈述、现有假设、表面现象） |
| Process | ① 识别表面现象 ② 剥离类比假设 ③ 追问底层不可再分要素 ④ 重新基于第一原理推导 |
| Output | 基于底层逻辑的解决方案（不依赖类比的原创方案） |
| Tools | SBR 模型、归纳-演绎闭环、因果链分析框架 |

## 示例：归纳的 IPO

| 维度 | 内容 |
|------|------|
| Input | 多个具体案例或观察数据 |
| Process | ① 收集案例 ② 识别共性模式 ③ 抽象为一般规律 ④ 验证规律适用范围 |
| Output | 从特殊到一般的一般性规律或原则 |
| Tools | MECE 法则、模式识别框架 |

## 与场景层的关系

场景的 composition 字段通过引用概念（concept://）来编排 IPO 流程。场景本身不定义 IPO，而是定义"在什么阶段、按什么顺序、调用哪些 IPO 方法"。
