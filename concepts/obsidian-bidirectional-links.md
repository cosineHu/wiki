# 双向链接（Obsidian）

Obsidian 最核心的功能，让笔记之间可以相互引用，构建网状知识结构。

## 为什么双向链接是革命性的

传统文件夹是**树状结构**，一个文件只能在一个位置。但知识是**网状的**——一个概念可能与多个领域相关。双向链接解决了这个根本矛盾。

## 链接语法

```markdown
[[笔记名]]           → 链接到同名笔记
[[笔记名|显示文本]]   → 自定义显示文本
[[笔记名#标题]]      → 链接到笔记中的特定标题
[[笔记名#^段落ID]]   → 链接到特定段落
```

## 链接类型

| 类型 | 语法 | 用途 |
|------|------|------|
| 标准链接 | `[[笔记名]]` | 链接到其他笔记 |
| 别名链接 | `[[笔记名\|别名]]` | 自定义显示文字 |
| 标题链接 | `[[笔记名#标题]]` | 跳转到特定标题 |
| 块链接 | `[[笔记名#^blockid]]` | 链接到特定段落 |
| 外部链接 | `[文本](URL)` | 链接到外部资源 |

## 反向链接的价值

每篇笔记底部自动显示**反向链接面板**——列出所有引用了当前笔记的其他笔记。这让知识网络双向可见：

- **正向链接**：从 A 链接到 B（"A 引用了 B"）
- **反向链接**：从 B 看到 A（"B 被 A 引用"）

## 嵌入笔记内容

```markdown
![[笔记名]]           → 嵌入整篇笔记
![[笔记名#标题]]      → 嵌入特定章节
```

嵌入让你在一篇笔记中直接看到另一篇笔记的内容，适合汇总页面。

## 踩坑提醒

- **不要只写不连**：每篇笔记至少关联一篇已有笔记，否则成为信息孤岛
- **链接比文件夹重要**：用链接表达关系，用文件夹做粗粒度分组
- **定期检查孤立笔记**：使用 Dataview 查询或脚本找出无链接的笔记

## IPO 建模

| 阶段 | 内容 |
|------|------|
| **Input** | Obsidian Vault 中的笔记，需要建立知识关联 |
| **Process** | ① 确定关联目标笔记 → ② 选择链接语法：标准链接 [[笔记名]] / 别名链接 [[笔记名|显示文本]] / 标题链接 [[笔记名#标题]] / 块链接 [[笔记名#^blockid]] → ③ 在正文中嵌入链接 → ④ 反向链接面板自动显示"谁引用了我" → ⑤ 可选：用 ![[笔记名]] 嵌入整篇内容 |
| **Output** | 网状知识结构：正向链接（我引用了谁）+ 反向链接（谁引用了我），知识网络双向可见 |
| **Tools** | Obsidian wikilinks 语法, 反向链接面板, 图谱视图 |
| **Quality Check** | 每篇笔记是否至少有一个链接？是否存在孤立笔记？图谱中是否出现了知识枢纽和聚类？ |
