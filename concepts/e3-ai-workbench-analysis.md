# E3 AI 工作台场景分析

基于 [[e3-ai-workbench|E3 AI 工作台项目]] 的系统性分析，提炼 AI 改造 E3OMS 的方法论和可复用模式。

## 项目定位

```
传统 E3+ 实施项目：系统替换/升级，关注功能覆盖
E3 AI 工作台项目：系统增强/AI 嵌入，关注降本提效
```

| 维度 | 传统 E3+ 实施 | E3 AI 工作台 |
|------|-------------|-------------|
| 目标 | 功能上线 | 效率提升 |
| 方法 | 调研→蓝图→开发→测试 | 痛点分析→场景筛选→AI 嵌入→效果度量 |
| 交付物 | 系统功能 | AI 能力 + 工作台 |
| 使用端 | E3OMS 中台 | E3OMS + 飞书双端 |
| 核心指标 | 功能覆盖率 | 操作步骤减少率、时间节省 |

## 专岗专治方法论

### 岗位痛点矩阵

```
岗位 × 痛点类型 × AI 解决方案
```

| 岗位 | 核心痛点 | 重复性操作 | AI 方案 |
|------|---------|-----------|--------|
| 运营 | 系统交互影响工作效率 | 库存捡漏计算、预售方案制定、数据看板制作 | 自动捡漏+预售方案+预测看板 |
| 客服 | 预售订单处理链路长 | 导表、核对、跨部门沟通 | 预售审单自动化 |
| 商品 | 建新品套餐压力大 | 多渠道多规格套餐手工创建 | 自动套餐创建 |

### 场景筛选原则

1. **高频重复**：日常高频操作，人工成本高
2. **规则明确**：业务规则可结构化表达
3. **效果可度量**：操作步骤减少可量化
4. **跨系统协同**：涉及 E3OMS + 飞书双端

## 一期 3.5 场景分析

### 场景 1：全链路预测分析看板

| 维度 | 说明 |
|------|------|
| **当前痛点** | 运营需手工汇总多维度数据制作看板 |
| **AI 能力** | 自动汇总 + 趋势预测 + 异常预警 |
| **提效点** | 看板制作从数小时→实时 |
| **使用端** | E3OMS + 飞书 |

### 场景 2：自动套餐创建&观测工作台

| 维度 | 说明 |
|------|------|
| **当前痛点** | 商品运营需为不同渠道手工创建不同规格套餐 |
| **AI 能力** | 基于盒规+渠道规则自动生成套餐（ZH/LG/DYCH） |
| **提效点** | 套餐创建从逐一手工→批量自动 |
| **使用端** | E3OMS + 飞书 |
| **依赖规则** | [[e3-auto-package-rules\|E3 自动创建套餐规则]] |

### 场景 3：库存捡漏+预售方案

| 维度 | 说明 |
|------|------|
| **当前痛点** | 运营需手工计算各渠道捡漏数量，人工制定预售方案 |
| **AI 能力** | 自动计算捡漏数量 + 智能推荐预售方案 |
| **提效点** | 捡漏计算从 30 分钟→实时，预售方案从半天→分钟级 |
| **使用端** | E3OMS + 飞书 |
| **依赖规则** | [[e3-channel-inventory-pickup\|渠道捡漏规则]] |

### 场景 3.5：预售审单自动化

| 维度 | 说明 |
|------|------|
| **当前痛点** | 客服导表→商品确认→人工筛选→人工占库存→人工审核 |
| **AI 能力** | 自动汇总→自动筛选→按时间序占库存→自动审核 |
| **提效点** | 6 步人工链路→全自动 |
| **使用端** | E3OMS + 飞书 |
| **依赖规则** | [[e3-presale-review-rules\|预售审单规则]] |

## 双端协同架构

```
飞书端（轻量交互）              E3OMS 端（深度操作）
├── AI 助手消息卡片              ├── AI 工作台嵌入
├── 看板推送                     ├── 套餐创建工作台
├── 预警通知                     ├── 捡漏数据面板
├── 审批流转                     ├── 预售审单工作台
└── 一键操作（确认/驳回）         └── 数据详情与配置
```

## 效果度量模型

| 场景 | 度量指标 | 基线（人工） | 目标（AI） |
|------|---------|------------|-----------|
| 预测看板 | 看板制作时间 | 2-4 小时/次 | 实时 |
| 自动套餐 | 单品套餐创建时间 | 5-10 分钟/SKU | < 10 秒/SKU |
| 库存捡漏 | 捡漏计算时间 | 30 分钟/次 | 实时 |
| 预售方案 | 方案制定时间 | 半天 | 分钟级 |
| 预售审单 | 单订单处理步骤 | 6 步 | 0-1 步 |
