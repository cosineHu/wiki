# DataMax 配补调退业务指标体系

基于 [[datamax|DataMax]] 平台 [[replenishment-allocation-transfer-return|配补调退业务]] 需求规格说明书提取的完整业务指标体系，覆盖补货、调拨、配货、退货四大模块。

## 一、补货指标

| 指标 | 计算方式 / 说明 |
|------|----------------|
| **安全库存** | ≤7天 = 上市库存；＞7天 = 每周一按近7天实际销量更新 |
| **上市库存** | SKU 首次到店量 |
| **上市天数** | 当前日期 - 上货日期 |
| **近 7 天平均销量** | 最近 7 天实际销量 / 7 |
| **目标库存** | 近 7 天平均销量 × X（可配置） |
| **目标库存上限** | 安全库存 + N（可配置） |
| **目标库存下限** | 安全库存 |
| **在仓库存** | 仓库当前可用库存 |
| **在途库存** | 已发货未到店库存 |
| **配未发库存** | 通知单已审核但未发出的待配货数 |
| **应补数量** | 目标库存 - 在仓库存 - 在途库存 - 配未发库存 |
| **补后库存** | 可用库存 + 补货量 |

## 二、调拨指标

| 指标 | 计算方式 / 说明 |
|------|----------------|
| **近 7 日销量** | 店铺最近 7 天销量 |
| **近 14 日销量** | 店铺最近 14 天销量 |
| **近 30 日销量** | 店铺最近 30 天销量 |
| **安全库存（动态）** | 7-14天无销 → 1件；21天无销 → 0 |
| **可用数** | 店铺当前可用库存 |
| **可调出量** | 可用数 - 安全库存（库存＞1时） |
| **库存保留数** | 调出后保留：近7日无销保留1件；近14日无销保留0件 |
| **调拨量** | 调入店铺的需求数量 |
| **调入后库存数** | 可用库存 + 调拨量 |
| **最近退（横调）日期** | 商品最近一次退仓/调出日期 |
| **最近到店日期** | 商品最近一次配送/调入日期 |

## 三、配货指标

| 指标 | 计算方式 / 说明 |
|------|----------------|
| **订单满足率** | 已配货量 / 订单需求量 |
| **尺码比 / 鞋码比** | 各尺码/鞋码的配货比例（如 S:M:L = 1:2:1） |
| **配货手数** | 增补款配货的分配单位 |
| **需求数量** | 订单或调入需求的数量 |

## 四、退货指标

| 指标 | 计算方式 / 说明 |
|------|----------------|
| **30 天不动销** | 近 30 天销量 = 0 |
| **45 天不动销** | 近 45 天销量 = 0 |
| **60 天不动销** | 近 60 天销量 = 0 |
| **可用库存数** | 退货量上限（退货量 ≤ 可用库存） |
| **退货量** | 默认 = 可用库存数，可手动调小 |

## 五、通用运营指标

| 指标 | 说明 |
|------|------|
| **累销数量** | 商品/店铺累计销量 |
| **定价（吊牌价）** | 商品原始定价 |
| **上货日期** | 商品首次到店日期 |
| **最近销售日期** | 最近一次销售日期 |
| **待发数量** | 通知单已审核但未发出的数量 |
| **波段** | 商品上市批次 |

## 六、指标全景

```
补货：安全库存 → 目标库存 → 应补数量 → 补后库存
调拨：滞销天数 → 可调出量 → 调拨量 → 调入后库存
配货：订单需求 → 满足率  |  尺码比 → 配货手数
退货：不动销天数 → 退货量（≤ 可用库存）
```

## 七、指标分类

| 类别 | 指标数 | 核心指标 |
|------|--------|---------|
| 库存类 | 8 | 安全库存、可用数、在仓库存、在途库存、配未发库存 |
| 销量类 | 5 | 近7/14/30日销量、累销数量、近7天平均销量 |
| 计算类 | 7 | 目标库存、应补数量、可调出量、调拨量、退货量 |
| 结果类 | 4 | 补后库存、调入后库存数、订单满足率、库存保留数 |
| 时效类 | 5 | 上市天数、上货日期、最近到店日期、最近销售日期、最近退日期 |
| 配置类 | 4 | 尺码比、配货手数、波段、定价 |

> **总计 30+ 个业务指标**，覆盖库存、销量、周转、满足率四个维度。
