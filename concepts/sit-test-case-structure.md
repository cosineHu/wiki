# SIT 测试用例结构分析

基于 [[sit-test-case-library|SIT 测试用例标准方案库]] 的结构性分析，提炼测试用例设计模式和可复用的测试方法论。

## 测试覆盖模型

### 五维测试覆盖

```
业务模块 × 测试角色 × 操作系统 × 业务场景 × 平台
```

| 维度 | 覆盖范围 |
|------|---------|
| 业务模块 | 2C订单 / 2B订单 / 全渠道 / 库存 / 对账（5 大模块） |
| 测试角色 | 客服 / 运营 / IT / 财务 / 物流 / 店员 / 系统（7 种角色） |
| 操作系统 | TMALL / 抖音 / 京东 / OMS / Adaptor / WMS / POS / SAP（8 个系统） |
| 业务场景 | 正向/逆向/异常/边界（4 类场景） |
| 平台 | 16 个电商平台 |

### 平台覆盖矩阵

每个平台需覆盖 5 大功能域 × 27 项功能点：

```
商品：初始化 → 下载 → 格式化 → 推送中台 → 库存同步
订单：下载 → 格式化 → 推送中台 → 发货回写 → 改址 → 备注下载/回写
退款：下载 → 格式化 → 推送中台 → 取消AG回写
退货：下载 → 格式化 → 推送中台 → 物流更新 → 入库回写
换货：下载 → 格式化 → 推送中台 → 同意换货 → 入库回写 → 发货回写
```

## 测试用例设计模式

### 1. 正向流程测试

验证核心业务流程的端到端正确性：

```
触发操作 → 系统响应 → 状态变更 → 数据验证 → 下游联动
```

**示例**（订单下载）：
```
客服下单 → Adaptor下载 → 格式化推送OMS → 生成零售订单 → 状态确认
```

### 2. 策略驱动测试

验证配置化策略的正确执行：

```
策略配置 → 触发条件 → 策略匹配 → 策略执行 → 结果验证
```

**示例**（分单策略）：
```
配置分单策略（区域仓库优先级）→ 客服下单 → 自动分单 → 匹配上海仓 → 库存占用验证
```

### 3. 多场景对比测试

同一功能在不同条件下的行为差异：

```
场景A（正常）→ 场景B（异常）→ 场景C（边界）
```

**示例**（赠品促销）：
```
买赠：买A → 自动赠B ✅
满赠（满足）：实付≥100 → 自动赠B ✅
满赠（不满足）：实付<100 → 不赠B ✅
```

### 4. 跨系统协同测试

验证多系统间的数据流转和状态同步：

```
系统A操作 → 推送到系统B → 系统B处理 → 回写系统A → 状态一致
```

**示例**（采购入库）：
```
SAP采购单 → E3+采购通知单 → WMS入库 → 回写E3+ → 同步SAP
```

### 5. 库存流水验证

所有涉及库存变动的测试都必须验证物理/逻辑两层流水：

```
操作 → 物理库存变动 → 逻辑库存变动 → 流水账核对
```

## 测试用例裁剪原则

### 按项目裁剪

| 裁剪维度 | 标准方案库 | 雅戈尔 E3+ 裁剪 |
|---------|-----------|----------------|
| 平台 | 16 个平台全覆盖 | 按雅戈尔实际渠道裁剪（淘宝/京东/唯品会/抖音/快手/自研商城等） |
| 采购管理 | SAP 对接 | 雅戈尔使用 SAP，保留；非 SAP 客户可裁剪 |
| 全渠道 | 网订店发/店订仓发/店订店发 | 雅戈尔有全渠道需求，全部保留 |
| 唯品会 | JIT/JITX 完整流程 | 雅戈尔有唯品会业务，保留 |
| 换货 | 淘宝/京东/小红书/苏宁 | 按实际支持的平台裁剪 |

### 按蓝图裁剪

```
业务蓝图大纲 → 识别需测试的功能模块 → 从标准方案库匹配用例 → 定制前置条件/测试数据
```

## 测试用例 → 蓝图 反向校验

测试用例应覆盖蓝图大纲中的所有功能点：

| 蓝图模块 | 标准用例覆盖 | 雅戈尔定制需补充 |
|---------|------------|----------------|
| 主数据 | 部分（通过其他用例间接覆盖） | 档案中心对接流程、唯一码管理 |
| 库存管理 | ✅ 采购/移仓/调拨/盘点/锁定/同步 | 雅戈尔四层库存模型专项测试 |
| 电商订单 | ✅ 下载/促销/合并/分单/审单/快递 | 抖音区域发货、GMV 分析 |
| 全渠道 | ✅ 网订店发/店订仓发/店订店发 | 即时零售分单、门店接单拒单 |
| 财务对账 | ✅ 配置/B2C对账 | B2B 对账、在途资金报表 |
| 技术架构 | ❌ 无覆盖 | 性能压测（729单/秒）、接口联通性 |
