# 电商业务调研大纲体系

[[baisheng-delivery-methodology|百胜价值交付方法论 V3.0]] 中的标准化调研大纲，覆盖电商业务实施的 6 大调研领域。每份大纲遵循"调研目的 → 调研重点 → 影响提示"三段式结构。

## 六大模块总览

### 1. IT 部门调研

**目标**：了解客户 IT 基础设施和系统架构。

| 调研项 | 核心问题 |
|--------|---------|
| 组织架构 | 贵公司的组织架构及其职责？ |
| 系统架构现状 | 当前的信息系统架构是怎样的？涉及哪些外部系统？ |
| 系统痛点 | 有哪些痛点希望通过新系统解决？ |
| 未来规划 | 希望未来的系统架构如何搭建？ |
| 调研安排 | 接下来的调研如何开展？ |

### 2. 基础主数据调研

**目标**：梳理组织、库存、配送等基础配置数据。

| 调研项 | 核心内容 |
|--------|---------|
| 组织架构 | 当前业务组织架构、未来规划 |
| 库存架构 | 仓库架构现状与痛点、E3+ 四层库存架构介绍 |
| 配送方式 | 快递/物流配送方式配置 |
| 营销区域 | 区域划分与营销策略 |
| 标签档案 | 商品/客户标签体系 |
| 售后原因 | 售后原因分类与标准化 |

### 3. 商品主数据调研

**目标**：明确商品属性定义、物料分类和档案管理。

| 调研项 | 核心内容 |
|--------|---------|
| 属性定义 | 基础属性、销售属性、辅助属性、扩展属性 |
| 物料类别 | 商品类别、商品类目 |
| 物料管理 | 物料档案、SKU 档案 |

### 4. 库存管理调研

**目标**：确定库存架构、对接方式和同步策略。

| 调研项 | 核心内容 |
|--------|---------|
| 仓库结构 | 当前仓库架构 |
| 库存对接方式 | 刷库存模式 vs 单据对接模式 |
| 库存同步 | 共享模式 vs 独享模式、同步规则 |
| 报表需求 | 当前报表使用情况、新增报表需求 |
| 接口清单 | 外部系统接口对接 |

### 5. 电商业务调研

**目标**：梳理 2C/2B/O2O 全业务流程。

| 调研项 | 核心内容 |
|--------|---------|
| 2C 正向订单 | 订单流程、促销策略、合并/拆分规则、仓库/快递适配、免审规则 |
| 2C 逆向流程 | 订单取消、退货退款、换货业务 |
| O2O 业务 | 线上订单门店发货/自提、门店退换货 |
| 售后流程 | 售后单据流转、退款策略 |
| 唯品会业务 | JIT/JITX 模式、档期管理 |
| 2B 业务 | 分销/经销订单流程、代发业务 |

### 6. 电商对账业务调研

**目标**：确定对账流程和财务报表需求。

| 调研项 | 核心内容 |
|--------|---------|
| 对账现状 | 当前对账流程、是否有系统支撑 |
| 基础配置 | 对账相关的基础参数配置 |
| 2C 对账 | 各电商平台对账流程 |
| 2B 对账 | 分销/经销对账 |
| 财务报表 | 财务对账报表需求 |

## 三段式结构

每份调研大纲的每个问题都遵循统一结构：

```
调研问题
├── 调研目的：为什么要问这个问题（明确调研目标）
├── 调研重点：需要关注和收集哪些信息（指导调研执行）
└── 影响提示：调研结果对后续实施的影响（关联下游工作）
```

## 调研执行流程

```
售前资料分析 → 选择调研模块 → 按大纲逐项调研 → 输出调研日志 → 确认调研结果
```
