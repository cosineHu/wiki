# 交付文档模板结构分析

基于 [[delivery-document-templates|交付中心文档模板库]] 的结构性分析，提炼模板设计模式和可复用的文档工程方法。

## 模板体系全景

```
交付文档模板
├── 调研前：调研大纲模板（三段式结构）
├── 调研后：业务调研报告模板（流程分析 + 改善点汇总）
└── 标准方案库（内容填充源）
    ├── 调研大纲体系（6 大模块）
    ├── 蓝图大纲标准方案库（6 大分类）
    └── SIT 测试用例标准方案库（7 Sheet）
```

## 模板一：调研大纲模板

### 设计模式

```
封面（品牌标识 + 方法论版本 + 模块名称）
    ↓
文档管理（元信息：版本/审核/分发/存档）
    ↓
目录（导航结构）
    ↓
调研问题 × N（每个问题：目的 → 重点 → 影响提示）
```

### 三段式结构分析

```
调研问题
├── 调研目的：WHY — 明确调研目标，让调研人员知道"为什么问"
├── 调研重点：WHAT — 指导信息收集，让调研人员知道"关注什么"
└── 影响提示：SO WHAT — 关联下游工作，让调研人员知道"影响什么"
```

**设计价值**：三段式结构确保调研不只是"问问题"，而是有目的、有重点、有产出的系统性工作。

### 影响提示的策略分层

| 策略类型 | 示例 |
|---------|------|
| 通用提示 | "为后续组织架构调研做铺垫" |
| 场景化提示 | "E3 升级 E3+ 项目：引导客户按业务召集用户快速推进" |
| 差异化提示 | "友商系统更换 E3+ 项目：着重了解当前系统功能，结合演示挖掘差异点" |

## 模板二：业务调研报告模板

### 设计模式

```
封面 → 文档管理 → 前言（感谢 + 保密） → 项目概述 → 流程分析 × N → 改善点汇总
```

### 流程分析四维模型

```
现有作业流程（AS-IS 可视化）
    ↓
流程说明（文字描述）
    ↓
现有问题（痛点识别）
    ↓
输出表单（交付物清单）
```

### 文档工程规范

| 规范项 | 说明 |
|--------|------|
| 版本管理 | 修改日期 + 版本号 + 描述 + 作者 |
| 审核批准 | 部门 + 姓名 + 签名 + 日期 |
| 分发控制 | 领取部门 + 姓名 + 签名 + 日期 |
| 存档策略 | 文档存档在出品方，项目组按需使用 |
| 保密声明 | 未经书面许可不得向第三方展示/阅读/传递 |

## 模板与标准方案库的关系

```
标准方案库（内容）  +  模板（格式）  =  项目交付文档
```

| 标准方案库 | 对应模板 | 产出文档 |
|-----------|---------|---------|
| 调研大纲体系 | 调研大纲模板 | 项目调研大纲 |
| 调研日志（项目） | 调研报告模板 | 项目调研报告 |
| 蓝图标准方案库 | — | 项目蓝图方案 |
| SIT 测试用例库 | — | 项目测试用例 |

## 模板裁剪原则

### 按项目类型裁剪

| 项目类型 | 调研大纲模板裁剪 | 调研报告模板裁剪 |
|---------|----------------|----------------|
| E3 升级 E3+ | 保留全部模块，重点在痛点和需求 | 流程分析侧重 AS-IS → TO-BE 对比 |
| 友商系统更换 | 增加系统功能对比调研 | 增加友商系统功能差异分析 |
| 新系统实施 | 保留全部模块，从零调研 | 流程分析侧重现状描述 |

### 按项目阶段裁剪

```
调研前：使用调研大纲模板 → 输出调研大纲
调研中：记录调研日志（结构化格式）
调研后：使用调研报告模板 → 输出调研报告
蓝图阶段：使用蓝图标准方案库 → 输出蓝图方案
测试阶段：使用测试用例标准方案库 → 输出测试用例
```
